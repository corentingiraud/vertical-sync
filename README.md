# Vertical Sync

A trail running training tool that pairs a **deterministic CLI** for FIT-trace
analysis with an **LLM coach** for plans, debriefs, and training memory.

It is built around a simple split: code computes the numbers, the LLM interprets
them, and your training memory lives in versioned markdown you own.

## The three layers

| Layer | Role | Where | LLM? |
|-------|------|-------|------|
| **Sense** | Deterministic FIT analysis, JSON output | CLI `vs` + `config/athlete.toml` | no |
| **Reason** | Interpretation, plans, debriefs, decisions | LLM via Claude Code skills | yes |
| **Remember** | Versioned markdown training memory | `coach/` | — |

The CLI never calls an LLM. It parses your `.fit` files and emits metrics and
plan comparisons (`--json` on every analysis command). The LLM reads that JSON
plus your training memory to interpret and decide — it is brought via
[Claude Code](https://claude.com/claude-code) skills (bring your own agent), not
an embedded API key.

This repo is a **public template**: all personal data is gitignored and rebuilt
from `templates/`. A fresh clone runs out of the box on example values.

## Install

Requires Python 3.12+ and [uv](https://docs.astral.sh/uv/).

```bash
uv sync
cp .env.example .env                                 # add your Coros or Garmin credentials
cp config/athlete.example.toml config/athlete.toml   # your physiology + race goal
```

`.env` holds your Coros or Garmin email/password (used only against the vendor's
own endpoints).
`config/athlete.toml` holds your HR max, HR zones, threshold pace, and goal race.

## CLI (`vs`)

All analysis commands support `--json` for structured, AI-readable output.

| Command | Purpose |
|---------|---------|
| `vs login [--source coros\|garmin]` | Test the Coros / Garmin connection |
| `vs download --start X --end Y [--source coros\|garmin]` | Download FIT files from Coros or Garmin |
| `vs list [--json]` | List available FIT files |
| `vs analyze <date> [--json]` | Analyze a single activity (YYYYMMDD or file path) |
| `vs profile <date> [--json]` | Pace + HR profile by gradient bucket |
| `vs week --start X --end Y [--json]` | Weekly summary + comparison to the plan |
| `vs assess [--start X --end Y] [--json]` | Strengths / weaknesses assessment |
| `vs plan [--week N] [--json]` | Show the training plan targets |
| `vs pdf` | Generate a print-friendly plan PDF |

## The coach (Reason + Remember)

Your training memory lives in `coach/` (gitignored). The schema is documented in
[`docs/coach-memory.md`](docs/coach-memory.md); start from `templates/`:

```
coach/
├── athlete.md           # persistent profile: strengths, weaknesses, injuries, gear, reference perfs
├── plan/YYYY-MM-DD.md   # one week each: TOML frontmatter targets (the CLI reads these) + prose
├── journal/YYYY-race.md # dated log, newest on top, one file per training block
└── races/<race>.md      # race-day cheatsheet + post-race debrief
```

Each plan week is a single markdown file: TOML frontmatter (the numeric targets
the CLI compares against) plus free prose for you and the coach. One source of
truth, no drift.

Two [Claude Code skills](.claude/skills/) orchestrate the loop:

- **`/debrief`** — run the CLI for the numbers, interpret them, write a dated
  journal entry, and promote durable facts into `athlete.md`.
- **`/plan-block`** — build or adjust the weekly plan files toward your race goal,
  reading your profile and journal.

## Disclaimers

- **Coros and Garmin APIs are unofficial.** This tool uses reverse-engineered
  Coros Training Hub endpoints (via [`coros_data_extractor`](https://github.com/gandroz/coros_data_extractor))
  and the Garmin Connect mobile endpoints (via [`garminconnect`](https://github.com/cyberjunky/python-garminconnect)).
  There is no official consumer API; endpoints may change or break, and use may
  be subject to each vendor's terms of service. Your credentials stay local in
  `.env` (Garmin session tokens are cached in `~/.garminconnect`).
- **Not medical advice.** Vertical Sync is a personal training aid, not a medical
  device. Any health or injury note (in the code, the templates, or generated by
  the LLM) is a non-medical hypothesis. Consult a qualified professional for
  diagnosis and treatment.
